# Visual Ranker — the one experiment that matters before building more

The riskiest assumption isn't "do people disagree" (they do). It's
**"will a group actually complete the collection loop"** — the no-server
architecture taxes exactly the multiplayer step the value depends on.
Test that, cheaply, before writing another feature.

## Two loops, two cadences (the honest answer on daily vs weekly)

**Daily — solo dogfood (you, ~2 min/day).**
Use it on one real decision you're already making. This is what's caught
every real bug so far. Zero coordination cost, so daily is right.
Log: did anything annoy you, break, or feel missing?

**Weekly — the real test (a group, ~1–2 real panels).**
You cannot assemble a fresh 5-person panel daily; recruiting is the slow
part, not the tool. So the group test is weekly, but structured to learn
fast within the week:

  Day 1  Pick a real decision with a real panel (a hiring loop, a grant
         committee, choosing a vendor). Build the board + rubric. Send
         each person the board file (or share link) and the rubric.
  Day 2-3  Collect their exported files yourself. **Log the friction
           precisely**: how many reminders, who couldn't open the file,
           where they got stuck. THIS is the load-bearing measurement.
  Day 4  Aggregate. Show the group the fracture (Group + Individual view).
  Day 5  Ask the two questions below.

## Measure exactly two things (don't over-instrument)

1. **Did surfacing a specific disagreement change the discussion or the
   outcome** vs. what the meeting would have done alone?
   Win = "huh, I didn't realize we split on X."  (Insight validated.)

2. **Did the collection loop complete without you hand-holding?**
   If you personally emailed everyone twice, the *architecture* is the
   bottleneck, not the idea.  (This is the kill-or-continue signal.)

## The decision tree it produces

- Insight lands + collection smooth  → build the disagreement-first view,
  rubric-as-object. The reframe holds.
- Insight lands + collection painful → the local-first constraint is
  strangling the loop. Solve collection (one link that gathers replies)
  BEFORE more visualization.
- Insight doesn't land → the reframe is wrong; stop, rethink the wedge.

Do not build the next feature until you have run this once with real people.
